# Opik LLM Observability Setup

This document explains how to set up Opik for LLM observability in the Agno Blog Post Generator.

## What is Opik?

Opik is an open-source LLM evaluation and observability platform by Comet ML that provides:
- **LLM Tracing**: Automatic tracing of all LLM calls and agent interactions
- **Performance Monitoring**: Track latency, token usage, and costs
- **Prompt Management**: Version and compare different prompts
- **Evaluation**: Built-in metrics for LLM output quality
- **Debugging**: Detailed logs of agent reasoning and tool usage

## Installation

Opik and its dependencies have been installed using:
```bash
uv pip install opik opentelemetry-sdk opentelemetry-exporter-otlp openinference-instrumentation-agno
```

**Note:** This integration follows the official Opik documentation for Agno:
https://www.comet.com/docs/opik/integrations/agno

## Configuration

### 1. Create .env file

Copy the `.env.example` file to `.env`:
```bash
cp .env.example .env
```

### 2. Add Opik Credentials

Edit the `.env` file and add your Opik credentials:

```bash
# Opik Configuration (LLM Observability)
OPIK_API_KEY=your_opik_api_key_here
OPIK_PROJECT_NAME=agno-blog-llm-tracing
OPIK_WORKSPACE=your_opik_workspace
```

**Note:** Opik uses the same Comet ML platform, so you can use the same API key and workspace as Comet ML.

### 3. Get Your API Key

1. Go to [Comet ML](https://www.comet.com/)
2. Sign in or create an account
3. Navigate to Settings → API Keys
4. Copy your API key
5. Add it to your `.env` file

## What Gets Traced

Opik automatically traces all LLM interactions in your Agno agents:

### Agent-Level Tracing
- **Agent Invocations**: Every time an agent is called
- **Agent Inputs**: The prompts and parameters sent to agents
- **Agent Outputs**: The responses generated by agents
- **Agent Metadata**: Model used, temperature, max tokens, etc.

### LLM Call Tracing
- **Model Calls**: Every call to the underlying LLM (Gemini)
- **Prompts**: Complete prompts sent to the LLM
- **Responses**: Full responses from the LLM
- **Token Usage**: Input tokens, output tokens, total tokens
- **Latency**: Time taken for each LLM call
- **Cost**: Estimated cost per call (if available)

### Tool Usage Tracing
- **Tool Calls**: When agents use tools (search, scraping)
- **Tool Inputs**: Parameters passed to tools
- **Tool Outputs**: Results returned by tools
- **Tool Errors**: Any errors during tool execution

### Workflow Tracing
- **Multi-Step Workflows**: Complete trace of multi-agent workflows
- **Agent Chains**: How agents call each other
- **Decision Trees**: Agent reasoning and decision paths

## How It Works

### Automatic Instrumentation

Opik uses OpenTelemetry to automatically instrument Agno following the official integration guide:

```python
from src.agno_blog.infrastructure.observability import instrument_agno_globally

# This instruments all Agno agents automatically
# Uses HTTP exporter and environment variables as per Opik docs
instrument_agno_globally()
```

**How it works:**
1. Sets `OTEL_EXPORTER_OTLP_ENDPOINT` to Opik's endpoint
2. Sets `OTEL_EXPORTER_OTLP_HEADERS` with authentication
3. Configures TracerProvider with HTTP exporter
4. Instruments Agno with `AgnoInstrumentor()`

Once instrumented, **all** Agno agent calls are automatically traced without any code changes.

**Technical Details:**
- Uses **HTTP exporter** (not gRPC) for better compatibility
- Uses **SimpleSpanProcessor** for immediate trace export
- Endpoint: `https://www.comet.com/opik/api/v1/private/otel`

### Integration Points

1. **Streamlit App**: Automatically instruments on startup
2. **CLI**: Can be enabled via configuration
3. **API**: Can be enabled for API endpoints
4. **Tests**: Can be enabled for testing and debugging

## Viewing Traces

### Opik Dashboard

1. Go to [Comet ML](https://www.comet.com/)
2. Navigate to your workspace
3. Select the "Opik" or "LLM Tracing" section
4. Select your project: `agno-blog-llm-tracing`
5. View traces with:
   - Timeline view of all LLM calls
   - Detailed prompt and response inspection
   - Token usage and cost analysis
   - Performance metrics and latency charts

### Trace Details

Each trace includes:
- **Span Tree**: Hierarchical view of agent → LLM → tool calls
- **Timing**: Duration of each operation
- **Inputs/Outputs**: Complete data flow
- **Metadata**: Model, parameters, environment info
- **Errors**: Stack traces and error messages

## Usage Examples

### Basic Usage (Automatic)

The tracing is automatic when using the Streamlit app:

```bash
streamlit run streamlit_app.py
```

All agent interactions are automatically traced!

### Manual Instrumentation

For custom scripts or notebooks:

```python
from src.agno_blog.infrastructure.observability import instrument_agno_globally
from src.agno_blog.application.blog_generation_service import BlogGenerationService

# Instrument Agno globally
instrument_agno_globally()

# Now all agent calls are traced
service = BlogGenerationService()
blog_post = await service.generate_blog_post(
    session_state={},
    topic="Your topic here"
)
```

### Context Manager Usage

For temporary instrumentation:

```python
from src.agno_blog.infrastructure.observability import OpikTracer

# Instrument only for this block
with OpikTracer() as tracer:
    # All agent calls here are traced
    service = BlogGenerationService()
    blog_post = await service.generate_blog_post(
        session_state={},
        topic="Your topic"
    )
# Instrumentation automatically removed after block
```

### Custom Configuration

```python
from src.agno_blog.infrastructure.observability import get_opik_tracer

# Create custom tracer
tracer = get_opik_tracer(
    api_key="your_api_key",
    project_name="custom_project",
    workspace="your_workspace"
)

# Instrument with custom config
tracer.instrument_agno()

# Your code here...

# Clean up
tracer.uninstrument_agno()
```

## Troubleshooting

### Tracing Not Working

**Problem**: Traces not appearing in Opik dashboard

**Solutions**:
1. Verify API key is correct in `.env`
2. Check that `OPIK_API_KEY` is set
3. Ensure `opik_enabled = True` in settings
4. Check console for error messages
5. Verify internet connectivity

### Import Errors

**Problem**: `ModuleNotFoundError: No module named 'opik'`

**Solution**:
```bash
uv pip install opik opentelemetry-sdk opentelemetry-exporter-otlp openinference-instrumentation-agno
```

### Instrumentation Errors

**Problem**: "Failed to instrument Agno"

**Solutions**:
1. Check that Agno is installed: `pip list | grep agno`
2. Verify OpenTelemetry packages are installed
3. Check for version conflicts
4. Try reinstalling: `uv pip install --upgrade opik openinference-instrumentation-agno`

### Performance Impact

**Problem**: Tracing slows down generation

**Solutions**:
1. Disable tracing for development: `settings.opik_enabled = False`
2. Use sampling (trace only some requests)
3. Batch span export is already configured for minimal impact

## Differences: Opik vs Comet ML

Both are part of the Comet ML platform but serve different purposes:

### Comet ML (Experiment Tracking)
- **Purpose**: Track experiments, metrics, parameters
- **Focus**: Model training, hyperparameter tuning, A/B testing
- **Data**: Metrics, parameters, artifacts, models
- **Use Case**: Compare different blog generation runs, track success rates

### Opik (LLM Observability)
- **Purpose**: Trace LLM calls, debug agents, monitor production
- **Focus**: LLM interactions, prompts, responses, token usage
- **Data**: Traces, spans, prompts, completions, tool calls
- **Use Case**: Debug agent behavior, optimize prompts, monitor costs

### When to Use Each

**Use Comet ML when you want to:**
- Compare different configurations
- Track overall success metrics
- Monitor cache hit rates
- Analyze performance trends

**Use Opik when you want to:**
- Debug why an agent made a decision
- Optimize prompts for better results
- Monitor token usage and costs
- Trace multi-agent workflows
- Inspect individual LLM calls

**Use Both for:**
- Complete observability of your LLM application
- End-to-end monitoring from experiment to production
- Comprehensive debugging and optimization

## Best Practices

### 1. Project Organization

Use descriptive project names:
```python
os.environ["OPIK_PROJECT_NAME"] = "blog-generator-production"
```

### 2. Tagging Traces

Add metadata to traces for filtering:
```python
# Traces automatically include agent metadata
# You can add custom tags via environment variables
```

### 3. Sampling

For high-volume production, consider sampling:
```python
# Trace only 10% of requests
import random
if random.random() < 0.1:
    instrument_agno_globally()
```

### 4. Error Handling

Always handle instrumentation errors gracefully:
```python
try:
    instrument_agno_globally()
except Exception as e:
    logger.warning(f"Failed to instrument Opik: {e}")
    # Continue without tracing
```

### 5. Cost Monitoring

Regularly check token usage in Opik dashboard to:
- Identify expensive prompts
- Optimize prompt length
- Monitor cost trends
- Set up alerts for high usage

## Security Notes

1. **API Key Protection**
   - `.env` file is gitignored
   - Never commit credentials
   - Use environment-specific keys

2. **Data Privacy**
   - All prompts and responses are sent to Opik
   - Ensure compliance with data policies
   - Can disable for sensitive data

3. **Access Control**
   - Workspace permissions in Comet ML
   - Team member access management
   - API key rotation
